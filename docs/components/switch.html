Switch

The Switch component is used to toggle between two binary states, such as On/Off, Enable/Disable, or Active/Inactive. It offers a direct, easy-to-understand interaction and provides instant feedback. The design ensures accessibility, responsiveness, and consistency across applications.

Anatomy

Variants

Default

This is a clean toggle without any accompanying label or icon. It's compact and ideal for minimal layouts where the switch's meaning is already clear from context.

With Symbol

This variant includes a symbol (like "|" for On and "â¯" for Off) inside the switch thumb. It enhances visual clarity and is useful when colour alone isn't enough to indicate the switch's state.

Properties

Label

The switch can include textual labels like Active/Inactive, improving state clarity for users, especially in forms and settings.

Label Alignment

The placement of the label can be customized to appear left or right of the switch. This helps maintain visual balance depending on layout needs and reading flow.

Interaction State

Default State

The standard, interactive state where users can toggle the switch.

Disabled State

The switch appears grayed out and is non-functional. Used when the option is not currently available.

Non-Editable State

The switch reflects a state but cannot be changed by the user. It may still appear active/inactive but is read-only.

Behaviours

Instant Feedback

Toggling the switch provides immediate visual and functional response, reflecting the new state clearly.

Responsive Layout

Adapts well across different devices and screen sizes without losing clarity or touchability.

Usage guide

Use Switches for Activation States

Use switches to represent binary on/off states. They are ideal for toggling settings or features that require immediate activation or deactivation.

Avoid using switches to indicate errors. Switches are not designed for error states.
